1. Sistemas Operativos Móviles y Arquitecturas

Android (Google): Basado en el kernel de Linux. Su arquitectura se divide en capas: Kernel, HAL, Librerías nativas/Runtime, Framework y Aplicaciones.

iOS (Apple): Basado en Unix (Darwin). Arquitectura Cocoa Touch: Core OS, Core Services, Media y Cocoa Touch.

2. Nativas vs Multiplataforma

  • Nativas: Desarrolladas específicamente para un SO (Swift para iOS, Kotlin para Android). Mayor rendimiento y acceso total al hardware.
  • Multiplataforma: Un solo código funciona en ambos (Flutter, React Native). Desarrollo más rápido pero a veces menor rendimiento.

3. Hardware de Dispositivos

Los componentes clave son: CPU (Arquitectura ARM), GPU, Memoria RAM (limitada), Sensores, Batería y Pantalla táctil.

4. Lenguajes de Programación

  • Android: Kotlin (oficial), Java.
  • iOS: Swift (moderno), Objective-C.
  • Multiplataforma: Dart (Flutter), JavaScript (React Native).

5. Entornos de Desarrollo (IDE)

  • Android Studio: Oficial para Android.
  • Xcode: Oficial para iOS (solo Mac).
  • VS Code: Popular para web y Flutter.

6. Diseño de Interfaces (UI)

Uso de controles (widgets) y diseño responsivo para adaptarse a múltiples pantallas.

7. Guías de Estilo

  • Material Design (Google): Superficies, sombras y tinta.
  • Human Interface Guidelines (Apple): Claridad, deferencia y profundidad.

8. Arquitecturas y Patrones

MVC, MVVM (Estándar moderno) y Clean Architecture.

9. Control de Versiones

Git y GitHub para gestión de código y trabajo en equipo.

10. Persistencia de Datos

Shared Preferences, SQLite, Room y Archivos.
